> Created: 2026-03-07 | Update this file as milestones are hit or the plan adjusts.

---

## Weekly Rhythm (Every Week)

| Day       | Focus                                                   | Time      |
| --------- | ------------------------------------------------------- | --------- |
| Monday    | DSA problem (new pattern)                               | 1–1.5 hrs |
| Tuesday   | Job applications (3–5 apps)                             | 1 hr      |
| Wednesday | System design concept or mini-mock                      | 1–1.5 hrs |
| Thursday  | DSA problem (reinforce same pattern)                    | 1–1.5 hrs |
| Friday    | Resume/LinkedIn polish OR behavioral prep OR apply more | 45 min    |
| Weekend   | Light review, commit memory to git, rest                | Optional  |

**Total:** ~5–6 hrs/week of focused prep.

## Red Flags to Watch (Adjust Plan If These Happen)

| Symptom                                      | Fix                                     |
| -------------------------------------------- | --------------------------------------- |
| Applying but no responses after 3–4 weeks    | Resume needs work — bring a bullet here |
| Getting screens but failing technical rounds | Accelerate DSA, add a mock/week         |
| Passing technicals but failing system design | More mock system designs, less DSA      |
| Feeling burnt out                            | Tell Claude — plan adjusts              |

---

## Key Principles

- **Referrals are 5–10x more effective** than cold applies — message 2–3 people in your network weekly
- **Apply to jobs you're 60–70% qualified for** — don't screen yourself out
- **One DSA pattern at a time** — fully internalize before moving on
- **System design is your hidden advantage** — real engineering experience translates; just need the vocabulary
- **Never stop the pipeline** until a signed offer is in hand
